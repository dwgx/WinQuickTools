# 工具规范

每个工具至少包含：

- 一个可直接运行的 `.bat`
- 一个 `README.md`
- 清楚的输入说明
- 清楚的成功、失败、跳过提示
- 风险说明
- 简单测试记录

## 命名

目录名使用英文，例如：

```text
tools/SetRunAsAdmin/
tools/FileNameTools/
tools/PathTools/
```

脚本名建议使用英文，避免下载或压缩后出现编码问题：

```text
SetRunAsAdmin.bat
CopyPaths.bat
CleanFileNames.bat
```

## 批处理要求

- `.bat` 必须使用 CRLF 换行
- `.bat` 不要带 UTF-8 BOM
- 第一行必须是 `@echo off`
- 支持路径带空格
- 尽量支持中文路径
- 高风险操作必须要求确认

## 输出格式

推荐格式：

```text
[成功] 已处理：路径
[失败] 原因：具体原因
[跳过] 不支持：路径

结果：成功 1 个，失败 0 个，跳过 0 个。
```

## 测试清单

每个工具至少测试：

- 普通英文路径
- 中文路径
- 带空格路径
- 多文件拖入
- 文件不存在
- 不支持的文件类型
- 无输入直接运行

## 安全原则

- 不默认删除用户文件
- 不默认修改系统级注册表
- 优先写入当前用户范围
- 修改前尽量保留已有设置
- 失败时显示具体原因

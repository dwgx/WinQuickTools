# WinQuickTools

一套面向 Windows 的便捷小工具合集，目标是把常见但繁琐的系统操作做成可以双击、拖拽、批量处理的脚本。

项目重点不是堆功能，而是把每个工具做得足够顺手：提示清楚、错误可读、支持中文路径、支持多文件拖拽，并尽量不要求用户手动打开复杂设置页面。

## 当前定位

- 系统小工具合集
- 主要面向 Windows 10 / Windows 11
- 优先使用 `.bat`、PowerShell、注册表和系统自带能力
- 工具默认做成“下载即可用”，尽量不依赖额外运行环境

## 已规划工具

| 工具 | 状态 | 说明 |
| --- | --- | --- |
| SetRunAsAdmin | 已有原型 | 批量把 `.exe` 或 `.lnk` 设置为“以管理员身份运行” |
| OpenHereTools | 规划中 | 快速在当前目录打开终端、管理员终端、资源管理器 |
| FileNameTools | 规划中 | 批量重命名、清理文件名特殊字符、追加日期 |
| PathTools | 规划中 | 复制文件路径、短路径、目录路径，支持多文件 |
| NetworkTools | 规划中 | 常用网络检查：ping、端口测试、DNS 查询 |
| CleanupTools | 规划中 | 清理临时目录、日志、空文件夹，带确认提示 |

## 工具设计原则

- 支持拖拽文件到脚本图标上运行
- 支持双击后拖入文件路径再回车
- 支持多个文件一次处理
- 支持中文路径、空格路径
- 输出中文提示，并区分成功、失败、跳过
- 不静默执行高风险操作
- 尽量保留用户已有设置，不做无关覆盖
- 每个工具都要有说明、风险提示和测试记录

## 目录结构

```text
WinQuickTools/
  tools/        具体工具脚本
  docs/         项目大纲、规范、路线图
  templates/    新工具模板
```

## 使用方式

每个工具会放在 `tools/<工具名>/` 下。进入对应目录后查看说明文档，再运行 `.bat` 文件。

## 风险说明

部分工具会修改注册表、快捷方式属性、文件名或系统临时文件。使用前请阅读对应工具目录里的说明。
